🌐 PART 1 · 대규모 협업 (Mass Collaboration)

1. 대규모 협업이란 무엇인가

디지털 기술과 고속 통신망의 발달 덕분에 일반 시민들이 과학적 연구와 데이터 수집에 직접 참여할 수 있게 되었고, 이를 시민 과학(Citizen Science) 또는 대규모 협업(Mass Collaboration)이라고 부른다.

이 방식은 마치 딥러닝 연산에서 CPU 한 개 대신 수백 개의 GPU를 병렬로 돌리는 것과 같은 원리로, 소수의 전문가만으로는 처리할 수 없는 방대한 양의 작업을 더 저렴하고 빠르게, 때로는 더 정확하게 수행할 수 있게 해준다.

핵심 인물: Matthew Salganik 이 방식을 다음 3가지 유형으로 분류함.

2. 3가지 유형 비교표

유형핵심 특징산출물대표 사례
① 인간 기반 연산
(Human Computation)		 거대 업무 → 마이크로 태스크로 분할 → 합산 평균값 Galaxy Zoo, 정치 성명서 코딩
② 공개 모집
(Open Call)		 해결책 공모 (해결 어렵지만 검증 쉬움) 단 하나의 최고 기여 Netflix Prize, Foldit, Peer to Patent
③ 분산 데이터 수집
(Distributed Data Collection)		 시민이 흩어져 데이터 수집 단순 합산 eBird, PhotoCity, 말라위 저널

3. ① 인간 기반 연산 (Human Computation)

단순하지만 규모가 너무 커서 소수의 전문가로는 감당하기 어려운 문제를 해결할 때 사용된다. 핵심 절차는 Split → Apply → Combine.

🛡️ 핵심 품질 관리 장치

① 작업 잉여성 (Redundancy): 동일한 작업을 여러 사람이 중복 수행하여 악의적 참여자·실수를 걸러냄.

② 가중 평균 (Weighting): 단순 평균이 아닌, 신뢰도 높은 평가자의 의견에 더 큰 가중치 부여.

🔬 사례 ① · Galaxy Zoo

천문학자들이 수백만 개의 은하 사진을 나선형/타원형으로 분류하는 작업을 혼자서는 끝낼 수 없었다. 일반 시민들에게 분류를 맡기는 프로젝트.

10만+자원자 수
90만+은하 이미지
평균 40명이미지당 중복 평가
4천만+총 분류 작업
100만 개ML 학습용 데이터
100억 개ML이 분류한 은하

💡 예상치 못한 발견: 처음엔 나선형/타원형만 분류하려 했으나, 자발적 토론으로 완두콩 은하(Pea Galaxy)라는 새 형태 발견 → Nature 게재

🔬 사례 ② · 정치 성명서 클라우드 코딩

정당 성명서의 문장이 좌/우 어디에 위치하는지 분류. 전문가는 비싸고 느림.

6번선거 분석
18개정당 성명서
~1,500명평가자 (MTurk)
20만개총 평가
문장당 5명중복 평가
5시간 / $360총 비용·시간

📊 일치도: 가중 평균값이 전문가 코딩과 매우 높은 일치도, 2개월 후 다른 데이터 재현 시에도 유지.

4. ② 공개 모집 (Open Call)

해결책을 만들기는 어렵지만 검증은 쉬운 문제에 적합. 목표는 가장 뛰어난 단 하나의 기여.

🎬 Netflix Prize

자사 추천 알고리즘 '시네매치(Cinematch)'보다 정확도 10% 이상 개선하는 알고리즘에 $1,000,000 (약 13억) 상금. 일부 평점을 숨겨두고 검증 → 영화와 무관한 분야 팀이 우승.

🧬 Foldit

단백질 접기 문제를 스크린세이버/게임 형태로 변환. 결과: 일반인들의 집단 지성이 최첨단 알고리즘보다 더 안정적이고 낮은 에너지 레벨의 해결책 발견.

⚖️ Peer to Patent

특허 심사관이 모든 선행 기술을 알 수 없는 문제. 자원자들이 선행 기술 탐색 → 전체 특허 신청의 약 30%에 달하는 부적격 특허를 걸러냄.

5. ③ 분산 데이터 수집 (Distributed Data Collection)

🦅 eBird

조류 관찰자들이 새의 위치/종류를 기록하는 DB. 탐조자 위치/희귀종 편향 발생 → 참여자 상호작용 + 통계적 가중치 보정으로 학술 논문 수준 품질 달성.

📸 PhotoCity

관광지 3D 모델 구현 위해 모든 각도의 사진 필요. 기존 사진과 겹치는 새로운 각도의 사진을 올리면 점수 추가 → 부족한 뒷면/측면 사진 유도 (주의 집중 유도의 좋은 사례).

6. 패시브 데이터 (Passive Data)

참여자가 적극적 의도 없이 자동으로 축적되는 데이터.

🚕 Pothole Patrol

도심 택시에 GPS + 가속도 센서 부착 → 주행 중 급격한 속도·위치 변화 데이터 수집 → 머신러닝 클러스터링으로 도로 파손 지점 파악. 택시 기사는 별도 노력 없이 그냥 운전만 해도 데이터가 자동 수집.

7. 대규모 협업 설계 5원칙

① 참여자 동기 부여 (Motivating Participants)

Amazon Mechanical Turk처럼 소액 금전 보상도 동기가 되지만, Galaxy Zoo의 공동체 의식이나 Foldit의 게임의 재미가 더 강력하게 장기 참여를 이끔. 개인 인센티브 + 집합적 가치를 결합.

② 이질성 활용 (Leveraging Heterogeneity)

참여자의 능력·성실도가 달라도 무작정 배제하지 않음. 위키피디아처럼 소수의 열성 기여자 (Fat head) + 가끔 참여하는 다수 (Long tail) 구조를 활용.

③ 주의 집중 유도 (Directing Attention)

PhotoCity의 점수 보상, Galaxy Zoo의 의견 분분 이미지 빈번 노출 등으로 참여자의 시선을 가장 필요한 데이터 쪽으로 유도.

④ 예상치 못한 결과 허용 (Allowing for Surprising Results)

Galaxy Zoo의 완두콩 은하 발견처럼, 처음 설계한 틀을 벗어난 뜻밖의 결과를 막지 않고 수용 → 혁신적 발견.

⑤ 윤리적 고려 (Ethical Considerations)

크레딧: 입자물리학 논문 기여자 전원 공저자 등재, Acknowledgment 명시.

프라이버시: Netflix 데이터처럼 익명화 데이터도 IMDb 등 외부 DB 결합 시 개인 식별 위험.

실전 팁: 100만 명 중 1명만 참여해도 전 세계 대상이면 절대 수는 충분히 큼. 처음부터 완벽하게 만들지 말고 프로토타입 → 빠른 테스트 → 발전 또는 빠른 철수 (스타트업 전략).

⚖️ PART 2 · 디지털 연구 윤리 (챕터 6)

1. 디지털 연구 윤리가 어려운 이유 — 이중 딜레마

⚠️ 두 가지 상반된 위험

① 무모한 위험: 프라이버시 침해·대중 감시 등 비윤리적 연구의 위험

② 엄격함의 함정: 2014년 에볼라 유행 당시 휴대전화 통화 기록 기반 역학 조사가 프라이버시 문제로 좌절된 사례처럼, 지나친 윤리 엄격함으로 유용한 연구가 무산

접근법의 차이

학문 분야접근 방식특징
사회과학·의학규칙 기반 (Rule-based)대학 IRB 사전 심사 필수
데이터·컴퓨터 사이언스사후 대처 / 컨퍼런스 중심사람 직접 대면 실험이 적음
디지털 연구 (제안)원칙 기반 (Principle-based)플랫폼 약관·법률·방법 변화 빠름

2. 3대 윤리 논란 사례

🔬 사례 ① · 페이스북 감정 전염 실험 (Emotional Contagion)

실험 설계: 약 70만 명의 사용자 뉴스피드에서 긍정적/부정적 단어를 임의로 삭제. 이후 사용자가 직접 쓰는 글의 단어 빈도 변화 관찰.

결과: 실험 효과 크기는 매우 작음.

비판: 코넬대 IRB 통과에도 불구 사전 동의 없이 감정 조작. PNAS 편집진도 공개적 우려 표명.

4원칙 위반: 인간 존중 / 선행(취약 집단 미보호) / 법과 공익 존중

🔬 사례 ② · 하버드 T3 (취향·유대·시간) 연구

실험 설계: 페이스북 사용자 데이터 + 대학 시절 기록을 결합 → 익명화 후 외부 공개. 하버드 IRB 통과.

비판: 익명화에도 불구 데이터 분석 시 대상 학교가 하버드임을 누구나 유추 가능. 졸업생들의 페이스북 사용 패턴·관계망·취향 등 민감 정보 노출.

교훈: 익명화만으로는 재식별(Re-identification)을 막을 수 없다.

🔬 사례 ③ · 앙코르 (Encore) 연구

실험 설계: 일부 웹사이트 협조로 소스 코드에 스크립트 삽입 → 방문자 컴퓨터가 자동으로 검열 의심 사이트에 접속 → 차단 여부로 검열 지도 작성. 170개국 이상 컴퓨터 동원. IRB 사전 승인 없음.

비판: 사용자가 모르는 사이 자신의 컴퓨터에 검열 사이트 접속 기록이 남음. 권위주의 국가 사용자는 정부 수사/처벌의 신변 위험에 노출.

3. 디지털 연구 윤리의 4가지 원칙

벨몬트 보고서(Belmont Report)의 의학 윤리를 디지털 시대로 확장한 4가지 원칙.

원칙핵심 내용위반 사례
① 인간 존중
Respect for Persons		 자율성 존중, 고지에 입각한 동의 (Informed Consent) 페이스북 실험, 앙코르
② 선행
Beneficence		 위험 최소화 + 이익 극대화. 오픈 액세스·코드 공개 페이스북(취약집단), 앙코르
③ 정의
Justice		 위험·이익의 공정한 배분. 사회적 약자 보호 앙코르(검열국가 시민)
④ 법과 공익 존중
Respect for Law and Public Interest		 컴플라이언스 + 책무성·투명성 앙코르(170개국), 페이스북

4. 두 가지 윤리 패러다임

결과주의 (Consequentialism)의무론 (Deontology)
철학적 배경벤담의 공리주의칸트의 윤리학
판단 기준결과의 효용 (사회적 이익)수단·절차의 정당성
대응 원칙선행의 원칙인간 존중 / Informed Consent
예시 입장"피해는 적고 효용은 크니 정당함""동의 없는 실험은 결과와 무관하게 잘못"
핵심 통찰: 두 패러다임은 종종 충돌함 (효용은 크지만 동의를 받을 수 없는 연구). 어느 하나를 절대적 기준으로 삼지 말고 맥락에 따라 균형을 찾는 실용적 태도가 필요.

📌 예시: "가짜 이력서 채용 차별 연구"

  • 의무론: 고용주가 사전 동의 없음 → 인간 존중 원칙 위반
  • 결과주의: 고용주 피해는 제한적, 사회 차별 규명의 공익 효용은 큼 → 정당화 가능

5. 4가지 까다로운 상황과 대응 방안

① Informed Consent가 불가능한 경우

채용 차별 실험처럼 사전 동의 시 연구 자체가 불가능. 동의를 연속적·단계적으로 접근.

  • 참가·이탈의 용이성: 검열 국가 등에서 언제든 쉽게 이탈할 수 있는 장치 마련
  • 제3자 동의: 학생 대신 학교가 동의 (학교가 부모 설득)
  • 사후 설명 (Debrief): 실험 후 상황 설명하고 동의 획득
  • 대표 표본 동의: 무작위 일부 표본에게만 제한적으로 동의
② 재식별 공격 (Re-identification) — 익명화의 한계

익명 데이터 + 외부 DB 결합 → 특정 개인 식별 가능.

  • GIC + 선거인 명부: 익명 의료 데이터를 우편번호·생년월일·성별로 결합 → 개인 식별
  • Netflix + IMDb: 익명 리뷰를 IMDb 공개 리뷰와 대조 → 사용자 식별
  • 뉴욕 택시 기록: 위치 데이터로 새벽 유흥 시설 이용자 추적
대응책 — 월드 가든 (Walled Garden): 완전 비공개(연구 불가)와 완전 공개(유출 위험) 사이에서, 안전한 방어벽 안에서만 제한적 접근 허용.
③ 프라이버시 — 맥락적 통합성 (Contextual Integrity)

프라이버시 침해 여부는 "공개된 곳에 있는가"가 아니라 데이터의 흐름(Flow)이 사용자가 기대한 맥락(Context)을 벗어났는가로 판단.

예시: SNS에 공개 게시물을 올린 것은 "친구들과의 일상 공유" 맥락. 연구팀이 "학술 연구 데이터" 맥락으로 사용 → 법적으로 공개 정보라도 맥락적 통합성 깨짐 = 프라이버시 침해
④ 불확실성 하의 결정 — 예방의 원칙의 함정

"위험할 수도 있으니 아무것도 하지 말자"는 예방의 원칙 (Precautionary Principle)만 고수하면 사회가 큰 비용을 치름.

  • 최소 위험 표준: 일상적 위험 수준과 비교하여 평가
  • 검정력(Statistical Power) 활용: 사전/사후 비교로 피험자 수 최소화
  • 파일럿 연구: 본 실험 전 소규모 윤리성 사전 검증
  • 단계적 실험 (Phased Trials): 코로나19 백신 임상 1상·2상·3상처럼 단계 확대

6. IRB는 윤리의 하한선이다

IRB 통과는 면죄부가 아니라 최소한의 기준선일 뿐이다. 페이스북 감정 전염 실험이 코넬대 IRB를 통과했음에도 세계적 비판을 받은 것이 이를 잘 보여준다.

연구자는 IRB 통과에 안주하지 말고, 논문 부록 등에 윤리적 노력을 투명하게 공개(Transparency)하고 그 정당성을 스스로 설명하는 책무성(Accountability)을 가져야 한다.

최종 결론적 조언

  • 윤리 문제를 이분법으로 보지 말 것 → 연속적 사고방식
  • 일반 시민의 감각과 규범을 고려할 것
  • 정확한 언어로 윤리 문제를 규정하고 사회적 합의(숙의) 능력이 필수
  • 시민사회는 데이터 활용 반대보다 감사(Audit) · 참여 · 공공 데이터 축적에 기여해야 함

평가자성향 점수 (1=극좌~10=극우)다른 문장 일치도
갑7높음
을7높음

정3낮음 (혼자 다름)

(1) 단순 평균값을 구하시오.
(2) 가중 평균 적용 시 어떤 평가자의 가중치를 낮춰야 하며 그 이유는?
(3) 이 방식이 전문가 코딩보다 장점은 무엇인가?

Q8

[데이터 해석] Galaxy Zoo 통계 분석:

  • 자원자 수: 10만 명
  • 분류 대상 이미지: 90만 개
  • 이미지당 평균 분류 횟수: 40회
  • 총 분류 작업 횟수: 4천만 회 이상

(1) Redundancy 없이 1인 1이미지로 배분했다면 10만 명으로 처리 가능한 이미지 수는?
(2) 실제로 40회 중복 평가를 한 이유는?
(3) 이후 ML로 확장했을 때 처리 가능해진 은하 수는?

(1) 10만 개 (1인 1이미지, Redundancy 없음). 90만 개 처리 불가.

(2) 단일 평가는 실수·편향·악의적 분류에 취약. 40명 중복 평가로 이상치를 걸러내고 신뢰도 높은 다수결/평균값을 도출. 품질 관리의 핵심 장치 = 작업 잉여성(Redundancy).

(3) 100억 개 은하 분류. 인간이 분류한 100만 개 데이터로 ML을 지도 학습 → 100억 개로 확장 → 인간 한계를 초월한 결과 달성.

② 코로나 감염 경로 파악을 위해 고위험군(노인·기저질환자)만을 대상으로 백신 없이 바이러스에 노출시키는 실험 설계

📝 서술형 (2문항)

Q10

[서술형] 넷플릭스 익명 데이터가 재식별 공격에 취약했던 이유를 설명하고, "월드 가든(Walled Garden)" 방식이 무엇인지 서술하시오.

모범 답안:

넷플릭스는 영화 추천 알고리즘 개선을 위해 익명화한 사용자 리뷰 데이터를 공개했으나, 연구자들이 이 데이터를 IMDb에 공개된 사용자 리뷰와 대조·결합하여 넷플릭스 사용자의 신원을 식별하는 데 성공했다. 이처럼 익명화 데이터도 외부 데이터와 결합되면 특정 개인을 알아낼 수 있는 재식별 공격(Re-identification Attack)에 취약하다. 유사한 사례로 GIC 의료 데이터 + 선거인 명부 결합, 뉴욕 택시 기록의 위치 정보 분석 등이 있다.

월드 가든(Walled Garden)이란 데이터를 완전 비공개(연구 불가)하거나 완전 공개(유출 위험)하는 극단 사이에서, 안전한 방어벽(접근 통제, 계약, 감사 절차 등)을 쳐서 제한적 조건 하에서만 데이터 접근을 허용하는 방식이다. 이를 통해 개인정보 위험은 최소화하면서도 사회적 연구 효용은 유지하는 타협점을 찾는다.
Q11

[서술형] 결과주의와 의무론의 관점에서 "가짜 이력서를 이용한 채용 차별 연구"를 각각 어떻게 평가하는지 비교하시오.

모범 답안:

① 의무론적 관점: 채용 과정에서 고용주는 연구의 피험자가 되었으나 사전 동의를 전혀 받지 못했다. 의무론은 결과와 무관하게 수단이 정당해야 하므로, 동의 없이 진행된 이 연구는 '인간 존중의 원칙'을 위반한 비윤리적 연구로 판단된다 (칸트 윤리학).

② 결과주의적 관점: 고용주가 받는 피해는 가짜 이력서 검토 시간 정도로 제한적이다. 반면 이 연구로 사회 구조적 채용 차별을 실증적으로 밝혀 정책 개선에 기여하는 공익적 효용은 훨씬 크다. 따라서 결과주의적으로는 비용 대비 이익이 명백히 크므로 윤리적으로 정당화 가능 (벤담 공리주의).

③ 결론: 두 패러다임이 충돌하는 이 경우, 연구자는 어느 하나만을 절대적 기준으로 삼지 않고 원칙들 간의 균형과 맥락을 고려하여 판단해야 한다. 동의의 연속적 접근(예: 사후 디브리프), 위험 최소화 장치 마련 등 실용적 타협이 필요하다.
